        def read_wellHead(data_dir, w_par):
    """
    Read well heads from a csv file
    well_head: file address
    props {'latlon_proj', 'xy_proj'}   


     - If the head_type to read is database 'db', min_long, max_long, min_lat, max_lat from w_params will be read
     - If the head_type is 'csv' then the file specified in the head_file is read
    """
  • Reads w_par['head_type']
  • If the head type is db, then it pulls data from EDW, otherwise it reads the data from whatever file is defined in w_par['head_file']
  • If X isn’t found in the dataframe extracted from the head file, then it assumes the coordinates are lat long and performs the conversion.
  • Default lat/long projection is EPSG:4267
  • Default XY Projection is EPSG:32140

        def read_wellPath(head_DF,data_dir, path_type,path_files,kb_def):
"""
Read files or dataframes, well by well and process them

head_DF - Well head data frame extracted above
data_dir - Directory where data (files) lives
path_type - w_par['path_type'] 
path_files - w_par['path_files']
kb_def - w_par['kbh']
"""
  • path_type can be db, txt, dat, dev, or csv
    • Text “txt” files are CSV files with more than one well
    • CSV “csv” files are well by well, per Mo’s comment
      # Read all from one csv file with txt extension to avoid confusion with well by well csv files
    • Well list used is the wells found in the header dataframe
    • Gets dataframe from specified file–be it EDW, CSV, DEV, DAT, or TXT.
    • Gets dataframe either well by well or for the entire project.
    • For alpha release we are going to support only csv and dev files.
    • Related files depend on filename matching the well name, so file names will have to match the corresponding well name

"""
  head_DF: Well head dataframe

  w_par: well parameters file w_par['parameter name']
"""
  • Different top types are accepted as denote d by w_par['top_type']
    • CSV, DB, or Petrel
  • Top files use the parameters: ‘Well, Surface, X, Y, Z, MD’
  • Extract DataFrame from whatever file is specified in w_par['top_file']
  • If XY is not present (as in the case of the sample file I have) then XYZ coordinates are generated using Measured Depth and the Well Path
  • Tops have a check which checks if the well head dataframe is in the tops dataframe. If the top is not in the well head dataframe, then the value is stored in this head_DF.csv file.
